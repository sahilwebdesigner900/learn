⚙️ Git & GitHub Complete Setup Guide

Learn Git from zero to hero with step-by-step instructions

🎯 START HERE: If you're new to Git, follow this section from top to bottom. Every step builds on the previous one!

🚀 ZERO TO HERO: Complete Setup Journey

You'll go from "What is Git?" to pushing code to GitHub in this section. Follow every step!

STEP 1️⃣

Install Git on Your Computer

First, check if Git is already installed:

bash
# Check if Git is installed
git --version

# If you see a version number like "git version 2.40.0", ✅ Git is installed!
# If you see "command not found", download it below

If Git is not installed, download it:

  • Windows: Download from git-scm.com/download/win and run the installer
  • Mac: Copy this command: brew install git (requires Homebrew)
  • Linux (Ubuntu/Debian): sudo apt-get install git
  • Linux (Fedora): sudo dnf install git

After installing, verify it worked:

bash
git --version
# Should show: git version 2.x.x ✅
STEP 2️⃣

Configure Git (First Time Only)

Tell Git who you are. Your name and email will appear on every commit:

bash
# Set your name (replace "John Doe" with your actual name)
git config --global user.name "John Doe"

# Set your email (replace with your actual email)
git config --global user.email "john@example.com"

# Optional: Set default editor to VS Code
git config --global core.editor "code --wait"

# Verify your settings were saved
git config --list

# Should show your name and email in output ✅
STEP 3️⃣

Create GitHub Account & Setup SSH Keys

Part A: Create Free GitHub Account

  1. Go to github.com
  2. Click "Sign up" button
  3. Enter your email, create password, choose username
  4. Verify your email address
  5. Done! ✅ You now have a GitHub account

Part B: Generate SSH Key (Recommended for faster authentication)

💡 Why SSH? SSH keys are more secure and faster than typing password every time you push to GitHub. Once set up, you never need to enter your password again!
bash
# Generate SSH key (replace email with yours)
ssh-keygen -t ed25519 -C "your@email.com"

# When asked "Enter file in which to save the key":
# Just press ENTER (keep default location)

# When asked "Enter passphrase":
# Just press ENTER twice (no passphrase needed)

# Start SSH agent
eval "$(ssh-agent -s)"

# Add SSH key to agent
ssh-add ~/.ssh/id_ed25519

# Display your public key (copy this output)
cat ~/.ssh/id_ed25519.pub

Part C: Add SSH Key to GitHub

  1. Go to GitHub → Click your profile picture (top right)
  2. Click "Settings"
  3. Left sidebar → "SSH and GPG keys"
  4. Click green "New SSH key" button
  5. Title: "My Computer" (or your computer name)
  6. Key: Paste the SSH key you copied above
  7. Click "Add SSH key"
  8. Test the connection: Open terminal and run:
bash
ssh -T git@github.com # You should see: "Hi your-username! You've successfully authenticated" # This means SSH is set up correctly ✅
STEP 4️⃣

Create Your First Repository on GitHub

  1. Go to github.com/new (logged in)
  2. Repository name: Type "my-first-project"
  3. Description: "Learning Git and GitHub" (optional)
  4. Public or Private? Choose "Public" (visible to everyone)
  5. ✅ Check: "Add a README file"
  6. ✅ Add .gitignore: Select "Node" or appropriate language
  7. Click green "Create repository" button
🎉 Congratulations! Your first repository is created on GitHub!
STEP 5️⃣

Clone Repository to Your Computer

Copy the repository from GitHub to your local computer:

  1. Go to your GitHub repository
  2. Click green "Code" button
  3. Select "SSH" tab
  4. Click copy icon (copies the SSH URL)

Now clone it to your computer:

bash
# Clone the repository (paste the SSH URL you copied)
git clone git@github.com:YOUR-USERNAME/my-first-project.git

# Navigate into the project folder
cd my-first-project

# Verify remote is set up correctly
git remote -v

# Should show:
# origin  git@github.com:YOUR-USERNAME/my-first-project.git (fetch)
# origin  git@github.com:YOUR-USERNAME/my-first-project.git (push) ✅
✅ Success! Your repository is now on your computer!
STEP 6️⃣

Make Your First Edit & Commit

Edit the README.md file:

  1. Open "README.md" in your text editor
  2. Add some text like: "# My First Project - Learning Git"
  3. Save the file

Check what changed:

bash
# See current status git status # Output shows "modified: README.md" (in red) # See exact changes git diff README.md

Save changes to Git (add & commit):

bash
# Stage (prepare) all changes
git add .

# Commit (save) with a message
git commit -m "Update README with project description"

# Verify commit was created
git log --oneline

# Should show your commit in the list ✅
STEP 7️⃣

Push Changes to GitHub

Upload your commit to GitHub (first push):

bash
# Push changes to GitHub (first time) git push -u origin main # Output will show something like: # Enumerating objects: 3, done. # ... # To github.com:YOUR-USERNAME/my-first-project.git # * [new branch] main -> main ✅ # Future pushes (easier) git push origin maingit push origin main
