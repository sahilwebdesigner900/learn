⚙️ Git — Essential Commands & Other Means

Quick reference for the most useful Git commands, troubleshooting checks, and alternative tools (GUIs & IDE integrations).

NOTE: This pane focuses on core Git commands and practical workflows. It intentionally omits platform-specific hosting steps.

🔧 Common Git Commands (copyable)

BASIC

Repository setup & snapshot

bash
# Initialize a repo
git init

# Clone an existing repo
git clone 

# Check status
git status

# Stage files
git add         # single file
git add .             # all changes

# Commit staged changes
git commit -m "Short, clear message"
BRANCHING

Branch workflow

bash
# Create and switch to a branch
git checkout -b feature/my-feature

# List branches
git branch

# Switch branch
git checkout main

# Merge branch into current
git merge feature/my-feature

# Delete local branch
git branch -d feature/my-feature
HISTORY

Inspecting history & changes

bash
# Show commit history
git log --oneline --graph --decorate

# Show changes in working tree
git diff

# Show changes staged for commit
git diff --staged

# Show a single commit
git show
RECOVERY

Undo & recovery

bash
# Unstage a file
git restore --staged 

# Discard local changes in a file
git restore 

# Reset last commit but keep changes staged
git reset --soft HEAD~1

# Reset last commit and unstage changes
git reset --mixed HEAD~1

# Recover lost commits (use with care)
git reflog
git checkout

🔁 Syncing (generic)

These commands are generic Git operations; they work with any remote hosting service.

bash
# Add a remote (one-time)
git remote add origin 

# Fetch remote changes
git fetch origin

# Pull remote changes into current branch
git pull origin main

# Push local branch to remote
git push -u origin feature/my-feature

🛠️ Other Means: GUIs, IDEs, & Helpers

If you prefer visual tools or integrated workflows, these are widely used and free for many use cases:

  • Git GUI clients: GitHub Desktop; SourceTree; GitKraken (free tier)
  • IDE integrations: VS Code Git panel; JetBrains IDEs built-in VCS
  • Command helpers: tig (terminal UI), gh (GitHub CLI) — optional
  • Automation: Use CI/CD pipelines to run tests and checks on push

✅ Quick Checks to Verify Repo Health

  • Check status: git status
  • Check recent commits: git log --oneline -n 10
  • Check remote URLs: git remote -v
  • Check branch differences: git diff main..feature/my-feature

📱 WordPress — Useful Free Plugins & WP-CLI Commands

Handy plugin recommendations and WP-CLI commands to manage WordPress from the terminal. Use Git to version theme/plugin code and WP-CLI for site tasks.

Recommended Free Plugins

  • Yoast SEO — SEO optimization and content analysis
  • Wordfence Security — Firewall & malware scanning
  • UpdraftPlus — Backups to cloud storage
  • Elementor — Visual page builder (free core)
  • WP Super Cache — Simple caching for speed
  • Contact Form 7 — Lightweight contact forms
  • WooCommerce — E-commerce (free core)
  • Smush — Image optimization
  • Really Simple SSL — Easy SSL setup
  • Classic Editor — Restore classic editing experience

WP-CLI Quick Commands

bash
# Check WP version
wp core version

# Update core, plugins, themes
wp core update
wp plugin update --all
wp theme update --all

# Activate/deactivate plugin
wp plugin activate contact-form-7
wp plugin deactivate hello-dolly

# Export/import database
wp db export backup.sql
wp db import backup.sql

Versioning WordPress with Git

Best practices:

  • Keep wp-content/themes/your-theme and custom plugins under Git.
  • Do not commit wp-config.php with secrets; use environment variables or a sample file.
  • Use .gitignore to exclude uploads, node_modules, and vendor directories.
  • Use WP-CLI for database syncs and migrations; keep schema changes scripted.
