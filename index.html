üîß Initial Configuration

Set up Git with your identity before your first commit:

bash
# Check Git version
git --version

# Configure global user name
git config --global user.name "Your Name"

# Configure global email
git config --global user.email "your@email.com"

# Set default editor to VS Code
git config --global core.editor "code --wait"

# Edit all configurations
git config --global -e

# List all configurations
git config --list

üìä Check Status & View Logs

bash
# Check current status (verbose)
git status

# Check status (short format)
git status -s

# View commit history
git log --oneline

# View commit history with graph
git log --oneline --graph

2Ô∏è‚É£ Git Initialization

Init: Used to create a new Git repository in your project

bash
# Initialize a new repository
git init

# Add remote origin (link your repo)
git remote add origin <repo-link>

# Verify remote is linked
git remote -v

# Check current branch
git branch

# Rename branch (master to main)
git branch -M main

# Push repository for first time
git push origin main

# Push with upstream tracking (one time)
git push -u origin main

3Ô∏è‚É£ File States in Git

Untracked: New file that Git doesn't yet track
Modified: File has been changed but not staged
Staged: File is ready to be committed
Unmodified: File unchanged from last commit

4Ô∏è‚É£ Add & Commit Changes

Add: Adds new or changed files from working directory to staging area

Commit: Record of change with a message

bash
# Add all files to staging area
git add .

# Add specific file
git add <filename>

# Commit with message
git commit -m "Your commit message"

5Ô∏è‚É£ Push Command

Push: Upload local repository content to remote repository

bash
# Push to main branch
git push origin main

# Push with upstream (first time)
git push -u origin main

6Ô∏è‚É£ Git Branches

Branches: Create separate development paths for features

bash
# Check current branch
git branch

# Rename branch (e.g., master to main)
git branch -M main

# Navigate to existing branch
git checkout <branch-name>

# Create and switch to new branch
git checkout -b <new-branch-name>

# Example: Create feature branch
git branch feature/navbar

# Switch branch (modern method)
git switch <branch-name>

# Delete branch
git branch -d <branch-name>

7Ô∏è‚É£ Git Stash (Save Work in Progress)

Save changes temporarily without committing when switching branches:

bash
# Stash current changes (save draft)
git stash

# Apply stashed changes
git stash apply

# Clear all stashed changes
git stash clear

8Ô∏è‚É£ Merging Code

bash
# Fetch new branches from remote
git fetch

# Compare two branches
git diff <branch-name>

# Merge branch into current branch
git merge <branch-name>

# Push merged code to remote
git push origin main

9Ô∏è‚É£ Pull Command

Pull: Fetch and download content from remote repo and update local repo

bash
# Pull latest changes from remote
git pull origin main

üîü Undoing Changes

bash
# Unstage specific file

# Undo last commit (keep changes)

# Hard reset - DELETE all changes (CAREFUL!)
git reset --hard HEAD~1

üöÄ GitHub Pages - Deploy Your Website

Host your website directly from GitHub - completely free!

Step-by-Step Deployment:

  1. Create a new repository
  2. Push your website files to repository
  3. Go to Settings ‚Üí Code and automation ‚Üí Pages
  4. Select Deploy from a branch
  5. Choose Main/Master branch + root folder
  6. Click Save
  7. ‚úÖ Your site is now live!
