üîß Initial Configuration

Set up Git with your identity before your first commit:

bash
# Check Git version
git --version

# Configure global user name
git config --global user.name "Your Name"

# Configure global email
git config --global user.email "your@email.com"

# Set default editor to VS Code
git config --global core.editor "code --wait"

# Edit all configurations
git config --global -e

# List all configurations
git config --list

üìä Check Status & View Logs

bash
# Check current status (verbose)
git status

# Check status (short format)
git status -s

# View commit history

# View commit history with graph
git log --oneline --graph

3Ô∏è‚É£ File States in Git

Untracked: New file that Git doesn't yet track
Modified: File has been changed but not staged
Staged: File is ready to be committed
Unmodified: File unchanged from last commit

4Ô∏è‚É£ Add & Commit Changes

Add: Adds new or changed files from working directory to staging area

Commit: Record of change with a message

bash
# Add all files to staging area
git add .

# Add specific file
git add <filename>

# Commit with message
git commit -m "Your commit message"

5Ô∏è‚É£ Push Command

Push: Upload local repository content to remote repository

bash
# Push to main branch

# Push with upstream (first time)

6Ô∏è‚É£ Git Branches

Branches: Create separate development paths for features

bash
# Check current branch
git branch

# Rename branch (e.g., master to main)
git branch -M main

# Navigate to existing branch
git checkout <branch-name>

# Create and switch to new branch
git checkout -b <new-branch-name>

# Example: Create feature branch
git branch feature/navbar

# Switch branch (modern method)
git switch <branch-name>

# Delete branch
git branch -d <branch-name>

7Ô∏è‚É£ Git Stash (Save Work in Progress)

Save changes temporarily without committing when switching branches:

bash
# Stash current changes (save draft)
git stash

# Apply stashed changes
git stash apply

# Clear all stashed changes
git stash clear
When you have uncommitted changes and want to switch branches, Git won't allow it. Stash saves these changes in a draft state - they won't be deleted or added to any branch, but you can retrieve them later!

8Ô∏è‚É£ Merging Code

bash
# Fetch new branches from remote

# Compare two branches

# Merge branch into current branch
git merge <branch-name>

# Push merged code to remote
git push origin main
Pull Request (PR): Instead of merging directly, create a PR on GitHub to notify teammates about changes. This allows code review before merging to main!

9Ô∏è‚É£ Pull Command

Pull: Fetch and download content from remote repo and update local repo

üîü Resolving Merge Conflicts

Merge conflicts occur when Git cannot automatically resolve differences in code between two commits. This usually happens when:

  • Two branches modified the same lines
  • One branch deleted a file another modified
  • Both branches added conflicting changes

Solution: Manually edit the conflicted files and mark as resolved, then commit

1Ô∏è‚É£1Ô∏è‚É£ Undoing Changes

Go back to previous saved points and revert changes:

Case 1: Undo Staged Changes

bash
# Unstage specific file
git reset <filename>

# Unstage all files
git reset

Case 2: Undo One Commit

bash
# Undo last commit (keep changes)
git reset HEAD~1

Case 3: Undo Multiple Commits

bash
# Reset to specific commit hash
git reset <commit-hash>

# Reset 2 commits back (mixed mode)
git reset -MIXED HEAD~2

# Hard reset - DELETE all changes (CAREFUL!)
git reset --hard HEAD~1

üöÄ GitHub Pages - Deploy Your Website

Host your website directly from GitHub - completely free!

Step-by-Step Deployment:

  1. Create a new repository (use simple name for your project)
  2. Push your website files to the repository
  3. Go to repository Settings
  4. Navigate to Code and automation ‚Üí Pages
  5. Under Build and Deployment
  6. Select Deploy from a branch
  7. Choose branch: Main or Master + root folder
  8. Click Save
  9. ‚úÖ Your site is now live! GitHub will provide you a URL
Your website will be available at: https://yourusername.github.io/repository-name
